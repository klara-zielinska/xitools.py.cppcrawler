Searching methods

The following example finds all methods containing health in their names, in all classes and structures of names Cv*Info across all source files located in the Sources directory.

01_search_health_methods\search_health_methods.py
1from xitools.cppcrawler import *
2crawler = CppCrawler("Sources")
3
4methSearchRe = r"(?i)\b(?P<name>\w*health\w*)\s*\‍("
5for src in crawler.loadSourceFiles(["*.h", "*.cpp"]):
6 if src.tryScopeToClassBody("class|struct", r"Cv\w*Info"):
7 found = []
8 for (rmatch, stag) in src.findAll(methSearchRe, scopeTag=True, skipBlocks=True):
9 if Syntax.parseMethPrototype(src.intCode(rmatch.start())):
10 found.append((rmatch, stag))
11
12 if found:
13 print("\n\n" + src.filepath() + "\n---")
14 for rmatch, (kind, name) in found:
15 print(f'({kind}) {name}::{rmatch.group("name")}', rmatch.startLoc())
16

The prefix (?i) in the regular expression stands for case insensitive, (?P<name>...) for a named capture group where name is the name, \b is a 0-width word edge, \w is a literal character and \s is a white space. (See the Python re module documentation for details.)

The method tryScopeToClassBody creates search scopes in the underlying object, one per each found class (note that structures and unions are also C++ classes). These scopes are tagged with pairs (kind, name), where kind is: class, struct or union and name is the class name.

The switch scopeTag in the findAll method specifies that each returned result should be paired with the tag of the scope where the match was found. The switch skipBlocks specifies to not search inside blocks.

Finding a methRe match does not guarantee, it is a valid method declaration (it could be an initializer). We verify this by calling parseMethPrototype, which except parsing also validate the prefix code.

The intCode method takes optionally start and end positions, and returns a piece of the internal code between them – no second argument means till the end. The internal code is the original code with all \<new line> sequences removed (same as the preprocessor does). The arguments are by default positions with respect to the original code. However the crawler can operate on both internal and original indices.

The start and startLoc methods of a match object return, respectively, the start position and the pair: line number, character position in the line, both with respect to original code.
